# Колоды контекста партии: Питч, Ресурс, Риск

Контекстные колоды задают сценарий каждой партии. По аналогии с «Бункером»: **Питч** = Катастрофа, **Условия** = Бункер, **Риск** = Угроза.

> В правилах колода по-прежнему называется «Ресурс стартапа» — это **условия работы** в команде, а не funding или MVP.

## Общая структура

| Колода | Карт | Когда используется |
|--------|------|-------------------|
| Питч | 12 | 1 карта в начале партии |
| Ресурс стартапа (условия работы) | 30 | 5 карт за партию (по 1 на раунд) |
| Риск | 24 | 5 карт за партию (по 1 на раунд) + финал в режиме «Запуск продукта» |

## Колода «Питч»

Задаёт **идею глобального приложения** — нереалистичного, фантастического, планетарного масштаба. Это не «ещё один SaaS», а безумный венчур в духе «а что если это сделать в App Store». Все аргументы кандидатов оцениваются относительно этой карты: как их IT-бэкграунд и навыки помогут **именно этому** продукту.

### Принципы колоды

| Принцип | Описание |
|---------|----------|
| Глобальный масштаб | Затрагивает планету, человечество или реальность в целом |
| Нереалистичность | Телепорт, Марс, общение с мёртвыми, работа во сне — норма |
| IT-обёртка | Всё подаётся как приложение / платформа / сервис |
| Контекст для дискуссии | Чем абсурднее питч, тем важнее умение связать обычные карты кандидата с безумием |

### Оси вариативности

| Ось | Варианты | Зачем |
|-----|----------|-------|
| Масштаб | Планета, солнечная система, метафизика | Меняет, чей бэкграунд «внезапно» релевантен |
| Тип безумия | Биотех, космос, время, сознание, логистика, экология | Разные партии — разные аргументы |
| Аудитория | Всё человечество, избранные, пост-апокалипсис | Влияет на ценность Активов |
| Стадия | Идея на салфетке, прототип в CERN, «уже в проде на 8 млрд» | Меняет ценность Достижений |

### Формат текста карты

```

```

## Колода «Ресурс стартапа» (условия работы)

Преувеличенные **условия работы** в стартапе. На карточке — коротко и сразу понятно. Открывается по одной на раунд в паре с Риском.

### Как читать карточку

| Поле на карте | Что писать |
|---------------|------------|
| **Заголовок** | 2–4 слова, суть условия |
| **РАЙ** или **АД** | метка в углу карты |
| **Текст** | одно предложение, до ~12 слов |

```

│  Работаешь в подвале.   │

```

## Колода «Риск»

Кинематографичные **нереалистичные угрозы** для стартапа. На карте — сюжет и удар по бизнесу. Игроки аргументируют, чьи **бэкграунд, навыки, достижения, сайд-проекты и активы** помогут пережить именно это.

### Как читать карточку

| Поле на карте | Что писать |
|---------------|------------|
| **Заголовок** | 2–5 слов, крючок сюжета |
| **Сюжет** | что случилось — 1–2 коротких предложения |
| **Бизнес** | как бьёт по стартапу — одно предложение |

Колонка **«Чем закрывают»** — только для ведущего и баланса колоды, на карту не печатается. Показывает, какие категории карт кандидата логично подставлять в аргументах.

| Сокращение | Категория кандидата |
|------------|---------------------|
| Б | Доменный бэкграунд |
| Н | Ключевой навык |
| Д | Достижение |
| С | Сайд-проект |
| А | Актив |
| Л | Личный факт |

**Пример (лицевая сторона):**

```
┌─────────────────────────┐
│                         │
│  CTO — голубь           │
│                         │
│  CTO выпил зелье и       │
│  превратился в голубя.   │
│  Улетел из окна.         │
│                         │
│  Бизнес: некому вести    │
│  архитектуру. Релиз —    │
│  вчера.                  │
└─────────────────────────┘
```

### Связь с аргументацией игроков

В круге питча после открытия Риска игроки должны явно связывать **открытые карты** с полями «Сюжет» и «Бизнес»:

- **Бэкграунд** — «я уже работал в похожем безумии» (regulated, highload, enterprise…)
- **Навык** — «я умею именно это чинить» (security, growth, system design…)
- **Достижение** — «я уже делал такое в реальном (IT) мире»
- **Сайд-проект** — «у меня есть площадка / репутация / OSS для этого»
- **Актив** — «я приношу ресурс, который закрывает дыру» (прототип, контакты, деньги…)

В финале «Запуск продукта» карта Риска релевантна, если ≥3 открытых карт кандидатов или условий работы логично закрывают **удар по бизнесу**.

## Пары Ресурс + Риск за партию

В начале партии формируется **5 пар** из перемешанных колод. Рекомендации при сборке колоды для печати:

1. Не дублировать одну ось (оплата / офис / график) во всех 5 картах партии.
2. Минимум 1 **рай** и 1 **ад** среди 5 открытых условий.
3. Питч должен **конфликтовать** с частью рисков и условий (иначе дискуссия плоская).
